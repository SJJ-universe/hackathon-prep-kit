# 06. 생활·안전 데이터 라이브 연결 (카탈로그 · 프록시 · 스냅샷)

가이드 02/05가 "어떤 데이터셋을 쓸지"를 ID로 알려준다면, 이 문서는 **그 데이터를 실제로 화면에 띄우는 구현층**이다. 당일 가장 자주 깨지는 세 지점(CORS · serviceKey 인코딩 · 라이브 실패)을 미리 막아 둔다.

> 반입 규칙: 여기 추가된 것은 전부 **범용 템플릿(프록시·fetch 헬퍼)** 과 **지식(카탈로그·가이드)** 이다. 주제 맞춤 앱 코드가 아니므로 반입 가능 범위다. 주제 로직은 여전히 당일 새 저장소에서.

---

## 왜 필요한가 (3가지 벽)

1. **CORS** — data.go.kr / safetydata / Kakao REST 는 응답에 `Access-Control-Allow-Origin` 이 없다. 브라우저가 직접 `fetch` 하면 차단된다. → 로컬 프록시가 중계한다.
2. **serviceKey 이중인코딩** — 포털이 Encoding/Decoding 두 형태로 키를 준다. 잘못 인코딩하면 `등록되지 않은 키(코드 30)` 로 20분을 태운다. → 프록시가 인코딩을 1회로 통제한다.
3. **라이브 실패** — 무대에서 키 한도/네트워크/빈응답은 늘 일어난다. → 사전 **스냅샷**으로 자동 폴백하고, 화면엔 "예시" 배지로 정직하게 표기한다.

---

## 파일 지도

| 파일 | 역할 |
|---|---|
| `data/catalog.json` | 생활안전 API 메타(엔드포인트·인증·파라미터·응답경로·함정). 기계가독 + 사람가독. |
| `scaffold/proxy.mjs` | 무설치 CORS 프록시 (Node 내장 모듈만). |
| `scaffold/proxy.py` | 동일 기능 Python 표준라이브러리판 (Node 없는 노트북용). |
| `scaffold/data.js` | 앱쪽 `DataLayer.load()` — 라이브→스냅샷 자동 폴백 + 출처(실시간/예시). |
| `tools/check-data.mjs` | 대회 전 키·CORS 점검 + 스냅샷 프리페치. `.ps1` 래퍼 동봉. |
| `.env.example` | 키 자리(복사해서 `.env`). `.env` 는 커밋 금지(.gitignore 처리됨). |
| `data/snapshots/` | check-data 가 만든 응답 스냅샷(라이브 폴백용). |

## 빠른 시작 — 집에서 (사전 준비 플랜)

```powershell
# 1) 키 채우기
Copy-Item .env.example .env      # DATA_GO_KR_KEY / SAFETYDATA_KEY / KAKAO_REST_KEY 입력

# 2) 사전 점검 + 스냅샷 확보 (어떤 API가 진짜 LIVE인지 표로 확인)
.\tools\check-data.ps1            # 또는: node tools/check-data.mjs

# 3) 프록시 띄우고
node scaffold/proxy.mjs           # 또는: python scaffold/proxy.py   (기본 8088)

# 4) 앱에서 호출 (당일 app.js 의 [입력→계산] 자리)
#    <script src="./data.js"></script> 로드 후:
#    const r = await DataLayer.load("kma-vilage-fcst", { nx:60, ny:127 });
#    const items = DataLayer.getPath(r.data, "response.body.items.item");
#    badge.textContent = DataLayer.sourceLabel(r.source);   // "실시간"/"예시"
```

`check-data` 판정: **LIVE**(성공+데이터) / **EMPTY**(0행, 파라미터·기간 조정) / **NOKEY**(.env 키 없음) / **IDTODO**(serviceId 미확정) / **FAIL**(오류).

## 노트북이 불분명할 때 (3단 안전망)

당일 제공 노트북에 무엇이 깔려 있을지 모른다 → 아래 순서로 자동 강등된다.

1. **Node 있음** → `proxy.mjs` + `check-data.mjs` (권장, 무설치).
2. **Python만 있음** → `proxy.py` 로 동일 동작. (스냅샷 저장은 node가 담당하니, 사전에 집에서 미리 떠 두는 걸 권장.)
3. **둘 다 없음 / 인터넷 불가** → 프록시 없이 **정적 스냅샷만**. `data/snapshots/` 를 served 폴더로 복사하면 `DataLayer` 가 `./snapshots/<id>.json` 으로 폴백한다(전부 "예시" 표기). 키·네트워크 0으로도 데모가 돈다.

`setup.ps1` 로 Node/Python을 까는 게 1순위지만, 위 강등 경로 덕에 **최악의 노트북에서도 화면은 나온다.**

## 핵심 함정 (카탈로그에도 박아둠)

| 함정 | 대응 |
|---|---|
| serviceKey 이중인코딩 | `.env` 에 **Decoding 키** 권장(프록시가 1회 인코딩). Encoding 키면 `DATA_GO_KR_KEY_ENCODED=true`. |
| safetydata 파라미터 | `serviceKey` **대문자 K**(camelCase) 여야 동작. |
| TAAS(KOROAD) | 문서는 `ServiceKey`(대문자 S). 실패 시 catalog 의 `auth.param` 을 `ServiceKey` 로. `siDo`+`guGun` 필수(전국일괄 아님). |
| HTTP 전용 호스트 | 1360000/1471000/B552061/1741000 은 http → https 페이지 직접호출 시 mixed-content 차단. 프록시가 해결. |
| 거의 전부 CORS 없음 | 브라우저 직접 fetch 불가. 유일 예외 = Kakao Maps **JS SDK** Geocoder(JS키+도메인등록, CORS 면제). |

## 데이터 사실 정정 (검증 2026-06-21)

- **민방위 대피소 15044951** = 사실 **파일(Excel) 데이터셋**(API 아님). API는 safetydata `DSSP-IF-00195`(확인됨) 또는 data.go.kr OpenAPI `15115459`/표준 `15021098`. 키 없이 즉시 쓰려면 15044951 파일을 동봉.
- **긴급재난문자**: data.go.kr `15134001` 은 LINK 유형 → safetydata(dataSn=228)에서 실등록·승인. 정확한 `DSSP-IF-#####` serviceId 는 **로그인 후 확인**해 `catalog.json` 의 endpoint(`__CONFIRM_SERVICE_ID__`)를 교체. 승인 심사가 있으니 **대회 전 미리 신청**.
- **즉시자동승인(라이브 사수 우선)**: 기상청 단기예보(15084084)·기상특보(15000415)·식약처 의약품회수(15059114)·TAAS(15057666 등)·무더위/지진해일 쉼터. 개발계정 10,000/일. 단 키 발급 후 **활성화 1~2시간** 지연 주의.
- `partial`/`unverified` 표시 항목(쉼터 응답형식·재난문자 파라미터명 등)은 catalog 의 `verified` 필드로 구분 — 코드 전에 라이브 1회로 확정.
