# 04. 에이전트 부트스트랩 — 새 PC에서 Claude Code(우선)·Codex 0부터 첫 실행까지

새 노트북이나 대회 제공 PC(Windows)에서 코딩 에이전트를 0부터 설치하고, 첫 실행에 성공시키는 단계별 가이드입니다. **제일 먼저 Claude CLI를 띄우는 흐름**을 1순위로 두고, 그다음 그 에이전트에게 `setup.ps1`을 맡겨 나머지 도구를 한 번에 까는 순서로 설계했습니다.

이 문서의 사실(패키지명·URL·최소 버전·인증 절차)은 2026-06 기준 웹 검색으로 확인했습니다. 확실하지 않은 항목은 본문에 `미확인`으로 표기했습니다.

---

## 한눈에 흐름

```
[0] 선행 확인        인터넷 / 관리자 여부 / winget 존재 / Node 18+ 설치 여부
        │            (Node 없으면 winget 한 줄로 설치)
        ▼
[1] Claude CLI 설치   (가장 먼저)
        │            권장: setup.ps1 원샷에 자동 포함
        │            단독: npm i -g @anthropic-ai/claude-code
        │            대안: 네이티브 설치 스크립트(irm ...install.ps1 | iex)
        ▼
[2] PATH / 실행정책   %APPDATA%\npm 이 PATH에 있나 / 새 터미널로 PATH 갱신
        │            PowerShell ExecutionPolicy(Bypass 또는 RemoteSigned)
        ▼
[3] 인증             A. 대화형 로그인(브라우저)  또는
        │            B. ANTHROPIC_API_KEY 환경변수(setx)
        ▼
[4] 검증·첫 실행      claude --version  →  프로젝트 폴더에서 claude 실행
        │
        ▼
[5] 나머지 전체 설치   에이전트가 이 저장소를 읽고 setup.ps1 실행
        │            (Git/Node/Python/VS Code/gh + serve/netlify-cli/Codex ...)
        ▼
[6] 저장소 열고 작업   동작 확인 → MVP 빌드 시작
```

핵심 원칙: **에이전트(Claude)부터 살린 뒤, 나머지 설치·세팅은 에이전트에게 시킨다.** 사람이 손으로 까는 도구는 Node와 Claude CLI 둘뿐이고, 그 이후는 `setup.ps1` 한 줄로 끝납니다.

## 0단계 — 선행 확인 (2분)

새 PowerShell 창을 열고 아래를 차례로 확인합니다.

| 확인 항목 | 명령 | 기대 결과 / 조치 |
|---|---|---|
| 인터넷 연결 | `Test-NetConnection api.anthropic.com -Port 443` | `TcpTestSucceeded : True`. 실패하면 사내 프록시/방화벽 문제(트러블슈팅 표 참조) |
| 관리자 여부 | `([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole('Administrator')` | `True`면 시스템 설치 가능. `False`여도 user 스코프로 대부분 진행 가능 |
| winget 존재 | `winget --version` | 버전이 나오면 OK. 없으면 아래 참고 |
| Node 설치/버전 | `node --version` | `v18.x` 이상이면 OK. 없거나 18 미만이면 아래에서 설치 |

winget이 없을 때: Microsoft Store에서 **App Installer**를 설치하면 `winget`이 생깁니다. Store 페이지 `https://apps.microsoft.com/detail/9NBLGGH4NNS1`. 설치 후 새 터미널을 엽니다.

### Node 설치 (Claude Code 필수 전제)

Claude Code는 npm 설치 경로를 쓰려면 **Node.js 18 이상**이 필요합니다(공식 npm 패키지 배지·설치 가이드 기준 "Node.js 18+"). 없거나 구버전이면 winget 한 줄로 LTS를 깝니다.

```
winget install -e --id OpenJS.NodeJS.LTS --accept-package-agreements --accept-source-agreements
```

설치 후에는 **새 터미널을 열어야** `node` / `npm`이 PATH에 반영됩니다. 확인:

```
node --version
npm --version
```

> 참고: 네이티브 설치 스크립트(아래 1단계 (c))를 쓰면 Node 없이도 Claude CLI가 동작합니다. 다만 이 키트의 `setup.ps1`은 npm 경로로 Claude를 깔므로 Node가 있어야 setup이 매끄럽습니다.

## 1단계 — Claude CLI 설치 (가장 먼저)

세 가지 경로가 있습니다. 상황에 맞게 하나만 고르면 됩니다. 공식 npm 패키지명은 **`@anthropic-ai/claude-code`**(확인됨)입니다.

### (a) 권장 — setup.ps1 원샷에 포함되어 자동 설치

이 키트의 `setup.ps1`이 Node·Git 등과 함께 Claude CLI를 npm으로 깔고 끝에 버전까지 검증합니다. 손이 가장 적게 갑니다.

```
powershell -NoProfile -ExecutionPolicy Bypass -File ./setup.ps1
```

- 장점: 도구 전체를 한 번에. 멱등(여러 번 실행해도 안전). 버전 검증 요약 출력.
- 단점: Node 등 다른 설치까지 같이 도므로 시간이 더 걸림. **인증은 자동으로 안 됨**(3단계 수동).
- 닭-달걀 주의: setup.ps1을 "에이전트에게 시키려면" 에이전트(Claude)가 이미 떠 있어야 합니다. 그래서 **첫 한 번은 사람이** 아래 (b) 또는 (c)로 Claude만 먼저 깔고 → Claude를 띄운 뒤 → 그 Claude에게 setup.ps1을 맡기는 흐름을 권장합니다.

### (b) 단독 설치 — npm (사람이 Claude만 먼저 깔 때 1순위)

Node가 이미 있으면 이 한 줄이 가장 빠릅니다.

```
npm install -g @anthropic-ai/claude-code
```

- 장점: 빠르고 단순. 버전 고정(`@latest`, 특정 버전 핀)에 유리. 사내 npm 미러와 호환.
- 단점: Node 18+ 필요. 전역 bin(`%APPDATA%\npm`)이 PATH에 없으면 `claude`를 못 찾음(2단계 참조). 업데이트는 수동(`npm i -g @anthropic-ai/claude-code@latest`).

### (c) 대안 — 네이티브 설치 스크립트 (Node 불필요)

Node를 깔기 싫거나 막힐 때. Windows PowerShell용 공식 원라인(확인됨):

```
irm https://claude.ai/install.ps1 | iex
```

(참고 — macOS/Linux는 `curl -fsSL https://claude.ai/install.sh | bash`)

- 장점: Node 의존성 없음. 백그라운드 자동 업데이트. 의존성 제로 설치.
- 단점: 설치 위치를 자체 user-local bin에 깔고 PATH에 추가하므로 **새 터미널**이 필요. 그 bin의 정확한 경로는 환경마다 다를 수 있음(`미확인` — 보통 사용자 프로필 하위). 사내 프록시 환경에서 스크립트 다운로드가 막히면 (b) npm 경로로 우회.

세 경로 요약:

| 경로 | 전제 | 업데이트 | 언제 |
|---|---|---|---|
| (a) setup.ps1 | (Node 자동설치 포함) | 수동 재실행 | 도구 전체를 한 방에 |
| (b) npm | Node 18+ | 수동(@latest) | Node 있음·가장 빠름 (사람 첫 설치 1순위) |
| (c) 네이티브 | 없음 | 자동 | Node 막힘·의존성 최소화 |

## 2단계 — PATH / 실행정책

### PATH: `claude`를 못 찾을 때

npm 전역 설치(`-g`)는 실행 파일을 **`%APPDATA%\npm`**(보통 `C:\Users\<사용자>\AppData\Roaming\npm`)에 둡니다. 이 폴더가 PATH에 없으면 설치는 됐어도 `claude` 명령을 못 찾습니다.

1. 가장 흔한 원인은 "설치한 그 창"이 옛 PATH라서 그렇습니다. **새 터미널을 열면** 대개 해결됩니다.
2. 현재 PATH에 들어있는지 확인:
   ```
   $env:Path -split ';' | Select-String npm
   ```
3. 없으면 user PATH에 영구 추가(새 터미널부터 적용):
   ```
   setx PATH "$($env:Path);$env:APPDATA\npm"
   ```
   (이미 들어있으면 중복 추가하지 말 것. setx는 현재 창에 즉시 반영되지 않으니 **새 창**에서 확인.)
4. npm이 보는 전역 위치 직접 확인:
   ```
   npm config get prefix
   npm root -g
   ```

네이티브 설치((c))로 깐 경우엔 설치기가 자체 bin을 PATH에 추가하므로, 마찬가지로 **새 터미널**을 연 뒤 `claude --version`을 확인합니다. 그래도 안 보이면 `npm` 경로((b))로 재설치하는 게 가장 확실합니다.

### PowerShell 실행정책 (ExecutionPolicy)

`setup.ps1` 같은 `.ps1` 스크립트는 기본 정책에서 차단될 수 있습니다. 두 가지 방법:

- 일회성(권장, 시스템 설정 안 바꿈): 스크립트를 실행할 때만 우회.
  ```
  powershell -NoProfile -ExecutionPolicy Bypass -File ./setup.ps1
  ```
- 영구(현재 사용자만, 안전한 절충): 서명된 원격 스크립트는 막고 로컬은 허용.
  ```
  Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
  ```

차이: **Bypass**는 아무 것도 막지 않음(해당 프로세스에만 한정해 쓰면 안전). **RemoteSigned**는 인터넷에서 받은 미서명 스크립트는 막고 로컬에서 작성한 건 허용 — 일상 개발용 영구 설정으로 적당합니다. 회사 정책으로 `Set-ExecutionPolicy`가 막혀 있으면, 일회성 `-ExecutionPolicy Bypass` 방식만 쓰면 됩니다.

## 3단계 — 인증

`claude`를 깔았다고 바로 동작하지 않습니다. 한 번 인증해야 합니다. 두 방법 중 하나를 고릅니다.

### 방법 A — 대화형 로그인 (구독 사용 시 권장)

`claude`를 처음 실행하면 브라우저가 열리고 Anthropic 계정으로 로그인하게 됩니다.

```
claude
```

- Claude Pro/Max 등 **구독**으로 쓰려면 이 방법을 쓰고, **`ANTHROPIC_API_KEY`는 설정하지 마세요.** 환경변수에 API 키가 있으면 Claude Code는 구독이 아니라 **API 종량제로 과금**합니다(공식 지원 문서 기준). 현재 인증 상태는 실행 중 `/status`로 확인합니다.
- 대회장 공용 PC라면, 끝나고 로그아웃(또는 `claude` 자격 정리)을 잊지 마세요.

### 방법 B — 환경변수 `ANTHROPIC_API_KEY` (API 계정/무인 실행 시)

구독 없이 API 키로 쓰거나, 로그인 창을 띄울 수 없는 자동화 환경이면 환경변수로 키를 박습니다. `setx`로 넣으면 **다음에 여는 터미널부터 영구** 적용됩니다.

```
setx ANTHROPIC_API_KEY "sk-ant-여기에-발급받은-키"
```

- 키 발급처: Anthropic Console — `https://console.anthropic.com` (API Keys 메뉴). 결제수단이 등록된 Console 계정이 필요합니다(무료 claude.ai 플랜으로는 Claude Code 사용 불가).
- 주의: `setx`는 **현재 창에는 반영되지 않습니다.** 넣은 뒤 새 터미널을 열고 `echo $env:ANTHROPIC_API_KEY`로 확인하세요.
- 주의: 위 방법 A(구독)와 충돌합니다. 구독으로 쓸 거면 이 변수를 비워 두세요. 제거는 `setx ANTHROPIC_API_KEY ""` 후 새 터미널(또는 시스템 환경변수 편집에서 삭제).

### 언제 무엇을?

| 상황 | 선택 |
|---|---|
| 내 노트북 + Claude 구독 | 방법 A(대화형) — 종량 과금 회피 |
| API 계정만 있음 / 사내 표준이 API 키 | 방법 B(`ANTHROPIC_API_KEY`) |
| 무인/자동화/CI | 방법 B |
| 공용 대회 PC, 잠깐 쓰고 정리 | 방법 A 로그인 → 종료 시 정리 |

### Codex 인증 (병행 시)

Codex CLI도 마찬가지로 둘 중 하나입니다.
- 대화형: `codex` 실행 후 "Sign in with ChatGPT" 선택(ChatGPT Plus/Pro 플랜 사용).
- 환경변수: `setx OPENAI_API_KEY "sk-..."` — 키는 OpenAI 플랫폼 `https://platform.openai.com/api-keys`에서 발급. (자세한 병행 설치는 7단계.)

## 4단계 — 검증 · 첫 실행

설치와 인증이 끝났으면 동작을 확인합니다.

```
claude --version
claude doctor
```

- `claude --version`이 버전을 출력하면 PATH·설치 OK.
- `claude doctor`는 설치/환경 상태를 점검해 줍니다(문제 있으면 원인 안내).

그다음 **프로젝트(이 저장소) 폴더에서** 에이전트를 띄웁니다.

```
cd C:\path\to\hackathon-prep-kit
claude
```

첫 프롬프트에서 동작을 확인하고(예: "이 저장소 구조를 한 줄로 요약해줘"), 정상이면 **나머지 전체 설치를 에이전트에게 맡깁니다.**

> 에이전트에게 줄 첫 지시(예):
> "이 저장소의 `README.md`와 `CLAUDE.md`를 읽고, 거기 적힌 대로 `setup.ps1`을 실행해서 나머지 도구(Git/Python/VS Code/gh, 그리고 serve·netlify-cli·Codex 등)를 전부 설치해줘. 끝의 검증 섹션에서 git/node/npm/python/gh/netlify 버전이 다 찍히는지 확인하고 NOT FOUND가 있으면 새 터미널 기준으로 다시 점검해줘."

`setup.ps1`이 끝나면 검증 요약에 `git / node / npm / python / gh / netlify / claude / codex` 버전이 나옵니다. 여기까지 오면 환경 구축 완료 — 빈 새 저장소를 만들고(`gh repo create`) MVP 빌드를 시작하면 됩니다.

## 5단계 — 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `claude : command not found` / 인식 안 됨 | npm 전역 bin(`%APPDATA%\npm`)이 PATH에 없거나, 설치한 창이 옛 PATH | 새 터미널 열기 → 안 되면 `setx PATH "$($env:Path);$env:APPDATA\npm"` 후 새 창. `npm config get prefix`로 실제 위치 확인 |
| `... cannot be loaded because running scripts is disabled` | PowerShell 실행정책 차단 | 일회성 `-ExecutionPolicy Bypass`로 실행, 또는 `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` |
| 설치/로그인 중 네트워크 타임아웃·SSL 오류 | 사내 프록시/방화벽이 외부 도메인 차단 | `$env:HTTP_PROXY` / `$env:HTTPS_PROXY` 설정, `npm config set proxy`·`https-proxy`. claude.ai·anthropic.com·registry.npmjs.org 허용 요청. 막히면 네이티브 대신 npm 경로 사용 |
| 응답이 401/인증 실패 또는 로그인 창이 안 뜸 | 키 미설정 또는 잘못된 키 | 방법 A로 `claude` 재로그인, 또는 방법 B로 `ANTHROPIC_API_KEY` 재설정 후 **새 터미널**. `/status`로 현재 인증 확인 |
| 예상치 못한 API 과금 / 구독이 안 쓰임 | `ANTHROPIC_API_KEY`가 설정돼 있어 종량제로 빠짐 | 구독으로 쓰려면 환경변수 제거(`setx ANTHROPIC_API_KEY ""`) 후 새 터미널, `/status` 재확인 |
| npm 설치는 됐는데 `claude`만 안 보임 | 전역 prefix가 비표준 위치 | `npm root -g` / `npm config get prefix` 경로의 상위 폴더를 PATH에 추가 |
| Node 구버전(18 미만)으로 설치 실패/경고 | 최소 Node 18 미충족 | `winget install -e --id OpenJS.NodeJS.LTS`로 LTS 설치 후 새 터미널. 또는 네이티브 설치((c))로 Node 우회 |
| `winget` 명령이 없음 | App Installer 미설치(구형 Windows 등) | Microsoft Store에서 App Installer 설치(`https://apps.microsoft.com/detail/9NBLGGH4NNS1`) 후 새 터미널. 정 안 되면 Node를 nodejs.org 설치본으로 수동 설치 |
| `setx` 했는데 현재 창에 반영 안 됨 | `setx`는 현재 세션 미반영(다음 세션부터) | 새 터미널을 열고 `echo $env:변수명`으로 확인 |
