# 00 사전준비 체크리스트 — 대회 전에 끝내는 것 (새 노트북 기준)

대회 당일 4시간은 "만드는 시간"이지 "설치·가입·승인 대기 시간"이 아니다. 이 문서의 항목은 **전부 대회 전에** 끝낸다. 당일에 남기면 안 되는 것: 도구 설치, 계정 가입, API 키 발급(승인 대기), 자산 다운로드, 도구 1회 예행.

이 PC가 곧 새 노트북일 수도, 대회 제공 PC일 수도 있다. **둘 다 같은 절차로 한 번씩** 통과시켜 둔다. 대회 제공 PC는 현장에서 처음 만지므로, 가능하면 사전 점검 시간이 주어질 때 이 문서로 그대로 훑는다.

## (a) 도구 설치 확인 — setup 실행 후 버전 체크리스트

가능하면 부트킷의 `setup.ps1`(있을 경우)을 먼저 돌려 일괄 설치·점검한다. PowerShell을 **관리자 권한**으로 열고 저장소 루트에서:

```powershell
# 실행정책이 막으면 현재 세션만 허용
Set-ExecutionPolicy -Scope Process -Bypass -Force
.\setup.ps1
```

`setup.ps1`이 없거나 일부만 잡는 환경이면, 아래 명령으로 **각 도구 버전을 직접 찍어** 표를 채운다. 한 줄이라도 "명령을 찾을 수 없음"이 나오면 그 줄은 미설치다.

```powershell
git --version
node --version
npm --version
npx http-server --version   # 없으면: npm i -g http-server
gh --version                # GitHub CLI. 없으면: winget install GitHub.cli
netlify --version           # Netlify CLI. 없으면: npm i -g netlify-cli
python --version            # 앱 프록시/로컬 서버용 (선택)
vercel --version            # 선택. 없으면: npm i -g vercel
```

설치 수단 메모: winget(`winget install Git.Git`, `OpenJS.NodeJS.LTS`, `GitHub.cli`)이 가장 간단. CLI 전역설치는 `npm i -g`. 대회 제공 PC에 전역설치 권한이 없으면 `npx <도구>`로 즉석 실행되는지 미리 확인해 둔다.

## (b) 계정 생성·로그인

각 계정은 **가입만으로 끝내지 말고, 해당 도구에서 로그인까지** 통과시켜 둔다(키 캐시가 PC에 남아 당일 재로그인이 빨라진다).

- [ ] **GitHub** 계정 생성·로그인. `gh auth login` 완료(브라우저 또는 토큰 방식). 확인: `gh auth status`
- [ ] **Netlify** 계정 생성·로그인. 무인증 Drop(드래그&드롭)만 쓰면 계정 없이도 되지만, CLI 배포(`netlify deploy`)와 사이트 관리·재배포를 위해 계정 권장. 확인: `netlify login` → `netlify status`
- [ ] (선택) **Vercel** 계정 생성·로그인. `vercel login`. Netlify가 막힐 때의 2차 호스팅.
- [ ] GitHub 신규 저장소를 **public**으로 만들 수 있는지 1회 확인(Pages는 무료 플랜에서 public repo 권장).

메모: 회사·학교 SSO 계정 말고 **개인 이메일 계정**으로 만든다(대회 PC에서 SSO 막힘 회피). 2단계 인증 백업코드를 휴대폰에 저장.

## (d) 도구 숙지 — 각 1회 예행 (당일 첫 실행이 되지 않게)

대회 당일 "처음 써보는 명령"이 하나도 없게, 빈 폴더로 아래를 **한 번씩** 굴려 본다.

- [ ] **정적 서버**: 아무 폴더에 `index.html` 하나 두고 `npx http-server -p 8080` → 브라우저에서 `http://localhost:8080` 열림 확인
- [ ] **Netlify Drop**: 그 폴더를 zip 또는 폴더째로 app.netlify.com/drop 에 드래그 → https URL 발급되고 열림 확인 (무인증 경로 손에 익히기)
- [ ] **Netlify CLI**(선택): `netlify deploy --dir . --prod` 동작 확인
- [ ] **gh + GitHub Pages**: 빈 저장소 1회 `gh repo create test-pages --public --source=. --push` → Pages 켜서 `https://<id>.github.io/test-pages/` 뜨는지 확인(반영에 1~2분)
- [ ] **로컬 폴백**: `ipconfig`로 IPv4 확인 → 같은 와이파이의 휴대폰에서 `http://<PC-IP>:8080` 접속 확인(방화벽 허용 프롬프트 한 번 "예")
- [ ] **QR 생성**: `npx qrcode "https://example.com" -o qr.png` 로 오프라인 QR이 만들어지는지 확인
- [ ] 예행으로 만든 테스트 저장소·사이트는 정리(삭제)해 둔다

상세 3경로 절차는 [`03_호스팅_심사위원_시연.md`](03_호스팅_심사위원_시연.md).
