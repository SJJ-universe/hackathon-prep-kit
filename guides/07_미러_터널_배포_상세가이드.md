# 07 미러·터널 배포 상세 가이드 — Vercel + cloudflared (현재 기준, 2026-06 검증)

[06 Netlify 배포](06_Netlify_배포_상세가이드.md)가 1순위(L0)라면, 이 문서는 그 **폴백 사다리의 다음 두 칸**이다. 라이브 데모의 제1원칙은 "네트워크는 반드시 한 번 고장난다"이고, 그래서 같은 빌드를 **다른 벤더에 한 번 더**(L1 Vercel 미러) 올려 두고, 둘 다 막히면 **내 노트북을 직접 공개**(L2 cloudflared 터널)한다. 무대에서는 만들어 둔 것으로 **전환만** 한다(즉석 제작 금지).

```
L0  Netlify(공개 정적·HTTPS)        <- 기본 (06 가이드)
L1  Vercel(다른 벤더 같은 빌드)      <- 한 호스트/CDN 장애·방화벽 차단 대비   (이 문서 1부)
L2  cloudflared 터널 + 내 폰 핫스팟  <- 공개 호스트 둘 다 불통/라이브 백엔드   (이 문서 2부)
L3  녹화 MP4(로컬·망 0)             <- 전부 불통 (03 가이드)
```

빠른 3경로 개요는 [03 호스팅·시연](03_호스팅_심사위원_시연.md)을 본다.

# 1부 — Vercel 미러 (L1)

## 1-1. 왜 "미러"인가

L0(Netlify)와 같은 산출물을 **서로 무관한 두 번째 벤더**(Vercel)에 한 번 더 배포해 둔다. DNS·CDN·장애 범위가 달라, 한쪽 벤더 장애나 대회장 방화벽이 `*.netlify.app`를 막아도 다른 URL은 살아 있다. 무대에는 **A/B 두 QR 카드**를 들고, A가 안 뜨면 B를 건넨다(수 초 전환).

## 1-2. 설치

```
npm install -g vercel
vercel --version
```
- 현재 버전대 **vercel 54.x**, **Node 18 이상**(키트 `setup.ps1`의 Node LTS면 충족).
- 명령 이름은 `vercel`(별칭 `vc`). 인자 없이 `vercel` = `vercel deploy`.

## 1-3. 인증 — 두 방식

대화형:
```
vercel login            # 브라우저/이메일
vercel login --github   # 깃허브로
```
비대화형(CI/에이전트): 토큰을 환경변수로.
```
# PowerShell
$env:VERCEL_TOKEN = "토큰값"
vercel --prod --yes --token $env:VERCEL_TOKEN
```
토큰 발급 경로(현재): **Account Settings → Tokens** (`https://vercel.com/account/tokens`). CLI로도 `vercel tokens add "이름"`.

## 1-4. 순수 정적 폴더 배포 (빌드 없음)

```
# 폴더 안에서
vercel                  # 프리뷰(미리보기) 배포 -> 임시 URL
vercel --prod           # 프로덕션 배포 -> 고정 *.vercel.app

# 완전 비대화형(프롬프트 기본값 수락)
vercel --prod --yes

# 폴더로 안 들어가고
vercel --cwd .\dist --prod --yes
```
- `--prod` 없으면 **프리뷰 URL**(라이브처럼 보여도 고정 도메인 아님). 미러용은 반드시 `--prod`.
- 프레임워크 없는 순수 `index.html`은 Vercel이 **그대로 정적 서빙**한다(별도 `vercel.json` 불필요).
- 첫 배포는 (`--yes` 안 쓰면) 팀 선택 → "기존 프로젝트 연결?"(N) → 프로젝트 이름 → 루트 → 빌드설정 오버라이드(N)를 물은 뒤, 폴더에 **`.vercel/` 링크 폴더**를 만든다. 이후 배포는 이를 재사용.

## 1-5. 고정/커스텀 URL

- 기본 URL: `https://<프로젝트이름>.vercel.app` — **프로젝트 이름이 곧 서브도메인**(선착순, 예약 불가).
- 이름 변경(서브도메인 변경): 대시보드 → 프로젝트 → **Settings → General → Project Name**.
- 커스텀 도메인: **Settings → Domains** → apex(A) 또는 서브도메인(CNAME). CLI: `vercel alias set <배포URL> <도메인>`.

## 1-6. 드래그앤드롭 "Vercel Drop"

- **`https://vercel.com/drop`** (2026-06-12 신설). 파일/폴더/zip을 드롭 → 팀 선택 → 프로젝트 이름 입력 → Deploy(바로 프로덕션).
- **로그인 필요**(팀·이름 선택 때문). 정적은 그대로, 프레임워크는 자동 빌드.
- 함정: **드롭할 때마다 새 프로젝트**가 생긴다(기존 프로젝트로 재배포 아님). 반복 갱신은 CLI를 쓰라.

## 1-7. SPA 라우팅 (새로고침 404 방지)

`vercel.json`(저장소 루트):
```
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```
단일 화면 정적이면 불필요. (키트 스캐폴드에 `vercel.json` 템플릿 포함.)

## 1-8. 무료 "Hobby" 한도 + 비상업 조항 (주의)

- 월 포함량(현재): Fast Data Transfer ≈ **100GB/월**, 배포 **100회/일**, 함수 호출 100만, 프로젝트 200개 등. 초과는 과금 대신 기능이 약 30일 일시정지(Hobby).
- **비상업 조항(원문):** "the Hobby plan restricts users to non-commercial, personal use only." 즉 Hobby는 **비상업·개인용**이다.
- 해커톤 회색지대: 수익 없는 데모/포트폴리오는 전형적 Hobby 용도지만, **스폰서·고객·수익화와 엮이면 "상업"으로 간주**되어 원칙적으로 Pro($20/월)가 필요하다. 미러 용도로 잠깐 쓰는 비수익 데모면 통상 문제 없으나, 상업성이 있으면 미러는 Netlify에 맡기고 Vercel은 피하거나 Pro로.

## 1-9. 함정 (2025–2026)

- **`.vercel/` 링크 폴더**는 `.gitignore`에 넣는다(첫 배포 시 생성). 다른 프로젝트로 재연결하려면 삭제하거나 `vercel link`. 스테일 링크는 엉뚱한 프로젝트로 조용히 배포한다.
- `--name` 플래그는 **폐지** → 프로젝트 링크/`--yes` 기본값 사용.
- **프리뷰 vs 프로덕션**: 맨 `vercel`은 프리뷰. 미러는 `vercel --prod`.
- 파일이 수천 개면 `vercel deploy --archive=tgz`.

# 2부 — cloudflared 터널 (L2)

## 2-1. 왜 "터널"인가

공개 호스트(L0·L1)가 둘 다 막히거나, **내 노트북에서 도는 라이브 백엔드**(프록시/LLM 서버)를 그대로 보여야 할 때, 내 노트북의 로컬 서버를 **공개 HTTPS 주소**로 즉시 노출한다. 핵심: **대회장 와이파이가 아니라 내 폰 핫스팟**에 노트북을 물려 자기 회선으로 서빙한다(대회장 와이파이는 데모 최대 사망원인).

> 왜 cloudflared인가(= ngrok이 아닌가): ngrok 무료는 브라우저 접속 시 **강제 "Visit Site" 경고 페이지**(인터스티셜)를 띄우고 **가입/계정**을 요구한다. cloudflared Quick Tunnel은 **무가입·무로그인·인터스티셜 없음**이라 심사위원 폰에서 바로 열린다.

## 2-2. 설치 (Windows)

winget(권장, ID 확인됨 `Cloudflare.cloudflared`):
```
winget install -e --id Cloudflare.cloudflared
```
직접 다운로드(패키지매니저 없이):
```
# 단독 실행 .exe (폴더에 두고 그 자리에서 실행, 또는 PATH 추가)
curl.exe -L -o cloudflared.exe https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-windows-amd64.exe
# 또는 MSI 설치본(PATH 자동 등록): .../cloudflared-windows-amd64.msi
```
- choco/scoop도 있음: `choco install cloudflared` / `scoop install cloudflared`.
- 주의: cloudflared는 Windows에서 **자동 업데이트 안 됨**(필요 시 수동).

## 2-3. Quick Tunnel 명령

```
cloudflared tunnel --url http://localhost:8137
```
- **계정·로그인·DNS 불필요.** 랜덤 `https://<랜덤>.trycloudflare.com` URL을 터미널에 출력(보통 10초 이내).
- 함정: `.cloudflared` 폴더에 **`config.yaml`이 있으면 Quick Tunnel이 시작되지 않는다** → 데모 때는 설정 파일 없는 상태로 실행.

## 2-4. 전체 흐름 (Windows, 터미널 2개 — 둘 다 열어 둬야 함)

```
# 터미널 1 — 빌드된 정적 폴더를 고정 포트로 서빙
npx serve ./dist -l 8137
#   (Node 없이:)  python -m http.server 8137 --directory dist
```
```
# 터미널 2 — 그 포트를 HTTPS로 공개
cloudflared tunnel --url http://localhost:8137
```
출력된 `https://<랜덤>.trycloudflare.com`을 심사위원에게 전달(QR). 터널이 **HTTPS**(실제 Cloudflare 인증서)로 종단하므로 **보안 컨텍스트가 성립** → 카메라(`getUserMedia`)·위치(`geolocation`)가 심사위원 폰에서 **작동**한다. (이게 `http://192.168.x.x` LAN 공유 대신 터널을 쓰는 결정적 이유 — LAN http에선 이 API들이 조용히 죽는다.)

## 2-5. 데모 내내 살아있게 유지

- **두 터미널 창을 끝까지 열어 둔다.** 하나라도 닫히면 서버(터미널1) 또는 공개 URL(터미널2)이 죽는다.
- 절전/화면꺼짐 차단(관리자 PowerShell, AC 전원):
```
powercfg /change standby-timeout-ac 0
powercfg /change monitor-timeout-ac 0
```
- **덮개 닫기 동작을 "아무 것도 안 함"**으로(제어판 > 전원 옵션 > 덮개를 닫으면). AC 전원 유지. (절전/종료 시 URL이 새로 바뀐다.)

## 2-6. 캐비엇 (Cloudflare 문서 기준)

- **테스트/개발용**: "Quick Tunnels are intended for testing and development only." 프로덕션 SLA 보장 없음.
- **휘발성 URL**: 프로세스가 살아있는 동안만 유효. 재실행하면 새 랜덤 URL.
- **동시성 한도 200 in-flight**: 초과 시 `429`. 심사위원 몇 명엔 충분하나 부하테스트 금지.
- **SSE 미지원**: Server-Sent Events(EventSource 기반 실시간) 안 됨. (WebSocket은 일반적으로 지원.)

## 2-7. 핫스팟·QR 메커닉

- 노트북을 **내 폰 핫스팟**에 연결(대회장 와이파이 회피). 터널 URL은 망과 무관하게 공개 HTTPS다.
- QR은 오프라인 생성: `npx qrcode "https://<랜덤>.trycloudflare.com" -o qr.png`. 단 터널 URL은 매 실행마다 바뀌므로 **현장에서 띄운 뒤 그 자리에서 QR 재생성**(미리 인쇄 불가).

## 키트 한 줄 (스캐폴드)

```powershell
# Vercel 미러 (프로덕션, 비대화형 가능)
.\deploy-vercel.ps1 -Dir .\dist
.\deploy-vercel.ps1 -Dir .\dist -Prod        # --prod --yes

# cloudflared 터널 (정적 폴더 서빙 + 터널을 한 번에)
.\tunnel-cloudflared.ps1 -Dir .\dist -Port 8137
```

## 폴백 사다리 — 전환 요약

| 층 | 트리거 | 전환 행동(수 초) |
|---|---|---|
| L0 Netlify | 기본 | A QR (06 가이드) |
| L1 Vercel 미러 | A URL 5초내 안 뜸/TLS경고/`*.netlify.app` 차단 | **B 카드** 건넴. `vercel --prod`로 미리 배포·예열 |
| L2 cloudflared | 공개 호스트 둘 다 불통/라이브 백엔드 필요 | 내 폰 핫스팟의 노트북 → 터널 URL/QR (즉석 재생성) |
| L3 녹화 MP4 | 전부 불통/시간부족 | 로컬 디스크 MP4 재생 (03 가이드) |

무대 전에 **L0·L1을 둘 다 예열**(10분 전 각 URL 1회 접속), **L2는 핫스팟 붙여 1회 리허설**, **L3는 오프라인 재생 확인**. 미검증 계층은 없는 것으로 친다.
