# 01 당일 런북 — 0~4시간 분 단위 (2인 분업)

전제: 사전준비([`00_사전준비_체크리스트.md`](00_사전준비_체크리스트.md))는 끝났다(도구 설치·계정·키·자산). 그러므로 당일 0~10분의 "환경셋팅"은 **설치가 아니라 clone/복사 + 키 주입**이다. 시제(구체 주제)는 시작과 동시에 공개된다고 가정한다.

역할 고정: **P1=코더**(스캐폴드·구현·배포·화면조작), **P2=기획/데이터/발표**(시제 재서술·차별성·더미데이터·카피·피치). 두 트랙은 가능한 한 병렬로 달리고, 정해진 머지포인트에서만 합친다.

알람을 미리 맞춘다: **+60분 / +150분 / +210분 / +225분(URL 확정) / +240분(제출)**.

- [ ] (P1) `.env` 생성 후 사전 발급 키 주입. `.gitignore`에 `.env` 포함 확인.

## 0:10–0:25 — 시제 분류 + 빌드카드 선택 (둘 다, P2 주도)

- [ ] (둘) 가까운 주력 블루프린트가 있으면 [`../07_blueprints/`](../07_blueprints)를 펴서 **구조만** 베이스로(코드 복붙 금지).

- [ ] (P2) 데이터 결정: **라이브 사수 1종**(자동승인 실데이터, 1순위 기상청 `15084084`)과 나머지(캐시 스냅샷+더미) 분리. 카탈로그에서 쓸 필드명·기준일자 필드 확정.

빌드카드 선정 원칙: 배점은 실작동 21·국민활용 19·문제정의 18.5가 본체(≈58점), AI 10. **AI 과투자 금지** — "AI를 빼면 무너지는 1장면"만 만든다([`../06_pitch/common-p0-checklist.md`](../06_pitch/common-p0-checklist.md) P0-6).

- [ ] (P1) **라이브 사수 API 1종을 프록시로 실호출**해 응답을 화면에 찍는다(라이브 증명 토대, P0-1). 키는 서버사이드에서만.
- [ ] (P2) 현실적 라벨의 **더미데이터 6건** 작성(실데이터 컬럼과 스키마 일치 — 나중에 실/더미 전환이 쉽게).

## 1:00–2:30 — 수직 슬라이스 2: 핵심 워크플로우 + AI 추론 + 출처칩

목표 머지포인트(+150분): 핵심 워크플로우 1회 정주행 + AI 추론 + 출처칩 작동.

- [ ] (P1) 핵심 입력→AI 추론→결과 경로 완성. **AI가 규칙으로 못 하는 1가지**(흐린 자연어 해석·비정형 기준↔조건 의미 정합·개체 해소)를 **원문 인용 스팬과 함께** 실시간으로(P0-6).
- [ ] (P1) **출처칩 3종 분리** 구현: [원문·기관·기준일자] / [본 서비스 가공·일반지침] / [예시·더미]. 추출값엔 원문 verbatim 인용 칩(P0-3).
- [ ] (P1) 라이브 증거 패널(모델명·지연 ms·응답 출처 링크)을 화면 한 구석에. "캐시가 아니라 방금 호출".
- [ ] (P2) 접근성 기본 ON 카피: 큰글씨·음성·쉬운말·브라우저 언어 자동감지(외국인). "모름 허용"(미선택=전체).
- [ ] (P2) 행동 출구 문구: 신청 링크·콜번호(129·119·1372·주민센터). 운영이양 한 줄: "정부24/복지로/안전디딤돌/소비자24에 얹는 모듈"(P0-4).

**+150분 게이트**: 심사위원이 새 입력을 부르면 실호출이 도는가? 출처칩이 갈려 있는가?

## 2:30–3:30 — 수직 슬라이스 3: 안전·degraded·로컬 검증

목표 머지포인트(+210분): degraded mode(망 끊김 폴백) + 환각 차단 + 더미/실 구분 표기 완성.

- [ ] (P1) **안전 오류 비대칭**(P0-2): 매칭 실패·저신뢰·캐시를 회색 '해당없음'이 아니라 **'확인 불가 → 사람에게 확인'**으로 강등. 안전 도메인은 불확실할수록 위험 한 단계 상향(단 침수 과대피처럼 상향이 더 위험한 경우는 비단정 배너).
- [ ] (P1) **degraded 시연 토글**: 망/한 API를 끊으면 캐시·보수동작·사람 연결이 뜨는 1컷.
- [ ] (P1) **로컬 검증**: `npx http-server`로 빌드 산출물을 띄워 클릭 정주행 + 콘솔 에러 0 확인. 빌드형(vite)이면 `npm run build` 후 `dist`를 서빙해 확인(배포 산출물과 동일 경로).
- [ ] (P2) 핵심 화면 **스크린샷 3장** + 정상 플로우 **30초 녹화**(시연 실패 백업, 규정 권장).
- [ ] (P2) [`../06_pitch/8min-runbook.md`](../06_pitch/8min-runbook.md)로 5분 동선·역할(P1 조작/P2 발표) 정리, 8대 킬러질문 답 준비.

**+210분 게이트**: degraded 뜨는가? 틀리게 입력하면 '확인 권고'가 뜨는가? 더미/실 칩이 갈려 있는가?

이 30분은 **새 기능 금지**. 배포 확정과 검증만 한다. 상세 절차는 [`03_호스팅_심사위원_시연.md`](03_호스팅_심사위원_시연.md).

- [ ] (P1) **배포 → URL 확정**: 1순위 Netlify Drop(폴더/zip 드래그→https, 무인증·즉시). 실패 시 GitHub Pages, 그래도 안 되면 로컬 폴백(`serve.ps1` 또는 `npx http-server` + 같은 와이파이/화면공유).
- [ ] (P1) **다른 기기에서 URL 열기**(휴대폰/다른 노트북) → 새로고침(캐시) → 핵심기능 1장면 동작 확인. QR 한 장 생성(`npx qrcode "<URL>" -o qr.png`).
- [ ] (둘) **공통 P0 6개 전부 Y**: 실호출 1회 / 안전 '확인 권고' / 출처칩 분리 / 흡수 모듈 / 75세·외국인 첫 결과 / AI 빼면 무너지는 1장면 → [`../06_pitch/common-p0-checklist.md`](../06_pitch/common-p0-checklist.md)
- [ ] (둘) 8분 리허설 **2회**: degraded 끊기는 "의도된 것"이라고 한마디. 모르는 건 "지금은 추정/더미"라고 정직하게.
- [ ] (P1) 제출: 작동 URL + (백업) 로컬 실행본 + 녹화 영상 + 스크린샷. `.env`·크레딧·한도 최종 확인.

**+240분 게이트**: 작동 URL을 다른 기기에서 방금 열어 핵심 장면을 확인했다 = 제출.

## 막힐 때 폴백 (멈추지 말 것)

| 막힌 곳 | 즉시 폴백 |
|---|---|
| 실데이터 API 승인 안 됨/응답 이상/한도 초과 | 사전 받아둔 **캐시 스냅샷 + 더미데이터**로 전환. 화면엔 "예시" 배지로 정직 표기. 라이브 사수 1종만 실호출 유지. |
| 좌표 변환 막힘(기상청 격자·대피소 도분초) | 시연 지역을 **고정 좌표 1곳**으로 하드코딩(변환은 데모 후 과제로 정직 언급) |
| LLM 호출 실패/지연/한도 | 프롬프트 1개로 축소, 결과 캐시 1건 준비. 그래도 안 되면 "AI 추론 결과(사전 캡처)"로 1장면만 보존 |
| 빌드(vite) 깨짐/시간 부족 | **정적 HTML 한 장**으로 강등(번들 없이 즉시 배포). 기능 1개만 남긴다 |
| Netlify Drop 실패 | GitHub Pages → 그래도 안 되면 **로컬 + 화면공유/같은 와이파이 QR** (시연 자체는 가능) |
| 호스팅 전부 막힘 | 로컬 `http://localhost`로 발표자 화면 직접 시연 + 30초 녹화 백업 재생 |

원칙: **선택지 앞에서 멈추지 않는다.** 가장 싼 폴백으로 즉시 내려가 "도는 1장면"을 지킨다. 한계는 숨기지 말고 칩·배너로 정직하게 표시한다(정직 자백은 감점이 아니라 신뢰).
