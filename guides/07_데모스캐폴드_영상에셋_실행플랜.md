# 07. 다음 작업 실행 플랜 — 데모 스캐폴드(#2) · 영상 에셋(#3)

전제: **당일 제공 노트북 사양 불명.** 그래서 모든 항목을 두 갈래로 준비한다.
- **플랜 A (현장 구현)** — 제공 PC가 쓸 만함(브라우저 + `setup.ps1` 로 Node/Python 설치 가능 + 인터넷 확인). 반입은 최소(범용 템플릿+지식)로, 당일 조립.
- **플랜 B (사전 완비)** — 제공 PC가 약하거나 오프라인 위험. 집에서 최대한 만들어 USB 번들로 들고 가서 당일은 콘텐츠 교체만.

공통 원칙: **제공 노트북에는 GPU/툴체인을 기대하지 않는다.** 무거운 생성(영상·이미지)은 전부 **집 PC(RTX 5070Ti)** 에서. 제공 노트북은 *브라우저로 덱 재생 + 데모 서빙 + 배포*만 하면 된다.

## 작업 #2 — 데모 스캐폴드 (생활안전 액션카드 UI, 주제 비종속)

### 무엇을 만드나
`scaffold/` 에 **주제 중립** 컴포넌트를 더해, 어떤 생활안전 데모든 30분 안에 얹게 한다(반입 규칙 부합 = 일반 템플릿).
- 액션카드 컴포넌트(`경보/위험 → 한 가지 행동` 카드: 제목·심각도·행동·거리/경로 슬롯)
- 결과 리스트 + 빈상태/로딩/에러 상태
- 출처 배지(실시간/예시) — `DataLayer.sourceLabel()` 연동
- 지도 placeholder + Kakao Maps JS SDK 임베드 헬퍼(좌표 마커) — 선택
- 취약계층 모드 토글(큰 글씨/고대비/음성 TTS `speechSynthesis`) 패턴
- 덱과 **공유 디자인 토큰**(deck-starter 의 `--bg/--safe/--alert/Pretendard`)을 `app.css` 변수로 정렬 → 슬라이드3 `<iframe>` 에 박았을 때 일체감

### 플랜 A — 현장 구현 (권장 기본값)
- 반입: 위 범용 컴포넌트 + `DataLayer` + `data/catalog.json` + 프록시(이미 킷에 있음) + **10분 배선 레시피**(아래).
- 당일: 새 저장소에 `scaffold/` 복사 → Claude가 액션카드에 `catalog id` 하나 배선 → 라이브 사수 1종(기상청) 실호출 + 나머지 스냅샷.
- 배선 레시피(요지): `index.html` 입력 → `app.js` submit 핸들러에서 `DataLayer.load(id,params)` → `getPath(r.data, responsePath)` → 액션카드 렌더 → 배지에 출처.
- 산출물: `scaffold/components/`(action-card.js/css, badge, a11y-mode), `guides/` 에 배선 레시피 1장.
- 게이트: 빈 PC에서 `serve.ps1`+`proxy.mjs` 로 카드가 스냅샷 데이터로 렌더되면 통과.

### 플랜 B — 사전 완비
- 집에서 **별도 저장소**(프렙킷 아님)에 리드컨셉(안심신호) 데모를 거의 완성: 스냅샷 프리페치, 덱 토큰으로 스타일, `demo_loop.mp4` 녹화.
- 당일: clone → 최종 숫자/문구 교체 → `netlify deploy`.
- **반입 규칙 경고:** 이건 "완성 주제 앱"이라 규정에 따라 반입 불가일 수 있다. 조직위에 (1) 사전 작성 코드 허용범위 (2) 별도 앱/에셋 반입 가부를 **명시 확인**. 불허면 즉시 플랜 A로. → 그래서 **플랜 A를 1순위로, B는 규정 확인 후 옵션.**

### 추정/순서
- 플랜 A 컴포넌트 제작: 반나절(집). 당일 배선: 30~45분.
- **#2는 슬라이드3(승부처)의 심장** → 먼저 진행 권장.

## 작업 #3 — 영상 에셋 배치 (집 PC, RTX 5070Ti) — 본질적으로 사전 제작

### 무엇을 만드나
deck-starter 5장 + 클로징에 맞는 **텍스트 없는 시네마틱 plate 6샷** I2V 루프.
샷리스트(재사용): ①밤의 한국 SF 도시 ②위험 b-roll(HUD 틀만) ③앱 UI 플레이트 ④데이터/파티클 모션 ⑤새벽 안전한 시민 ⑥로고 스팅.

### 파이프라인 (메모리의 Blackwell 함정 반영)
- 주력: **Wan 2.2 14B GGUF Q4 + Lightning**(야간 배치). 환경: CUDA 13.0 · Py3.13 포터블 · PyTorch ≥2.7 cu128 · **xformers 금지** · `--use-sage-attention` + Wan 조합은 **검은 화면** 유발 주의 · SageAttn 휠 cp313 불일치 주의.
- 대비: 클라우드 **Kling**(키퍼 빠름) / 무료 시네마틱 스톡(Pexels/Coverr/Mixkit).
- 인코딩: **H.264 MP4, 3~6초 루프**(프로젝터 하드웨어 디코드 우수). 한글 자막은 굽지 말고 덱 HTML 레이어로.

### 플랜 A — 린(현장 비중 ↑)
- 히어로 루프 **2~3개만** 사전 생성. 나머지는 덱의 **라이브 WebGL 신호장 배경**으로 커버(렌더 0). 제공 노트북이 약하면 WebGL을 이 사전 루프로 교체(룩 동일·리스크 0).

### 플랜 B — 풀(현장 비중 ↓)
- 6샷 전부 + 20~30초 하이프 릴(CapCut/DaVinci + Pixabay CC0 음악) 사전 완성, USB 번들.

### 추정/순서
- 집 GPU 야간 배치라 **#2와 병렬 가능**(서로 독립). 키퍼 1개당 15~30분 × 6샷 + 인코딩.
- 제공 노트북은 재생만 하므로 #3은 **노트북 사양과 무관** — 무조건 집에서 끝낸다.

## 당일 노트북별 결정 규칙 (30초 판단)

- **쓸 만함 + 인터넷 O** → 플랜 A: 현장 배선 + 라이브 사수 1종 + Kling 보너스컷.
- **약함 / 툴 설치 막힘** → 플랜 B: 사전 완성 데모/덱 재생 + 정적 스냅샷(키·네트워크 0) + 사전 영상.
- **오프라인** → 프록시 없이 `DataLayer` 정적 스냅샷 폴백, 전부 "예시" 배지. PDF+MP4 백업으로 발표 자체는 보장.

> 핵심: **A를 기본값으로 짓되, 모든 산출물이 B(완전 사전)에서도 그대로 돌아가게** 만든다. 그러면 어떤 노트북이 와도 무너지지 않는다.
