# CLAUDE.md — Claude Code 작업 규약

## 최초 실행 (필수)

이 저장소를 열면 가장 먼저 도구 전체를 설치한다. 다른 작업보다 우선한다.

```
powershell -NoProfile -ExecutionPolicy Bypass -File ./setup.ps1
```

설치 스크립트 끝의 검증 섹션에서 `git / node / npm / python / gh / netlify` 등 버전이
출력되는지 확인한 뒤에 작업을 시작한다. NOT FOUND 항목이 있으면 새 터미널을 열어
PATH 갱신 후 다시 확인한다.

## 반입 규칙

- 이 저장소는 도구 설치 / 일반 템플릿 / 지식만 담는다.
- 주제맞춤 완성 앱 코드는 담지 않는다. 대회 당일 빈 새 저장소에서 작성한다.
- `scaffold/` 는 아이디어 비종속 일반 템플릿 전용이다.

## 작업 컨벤션

- 스코프가 작은 변경을 선호한다. 무관한 파일은 건드리지 않는다.
- Grep/Glob 우선, 필요한 부분만 읽는다(넓은 파일 통독 지양).
- 기존 코드의 패턴과 구조를 따른다.
- 핵심 기능 1~2개에 집중하고, 동작하는 MVP를 호스팅(`netlify deploy` 또는 `vercel`)으로 마무리한다.

## 검증 후 완료 보고

- "완료/수정됨/통과"라고 말하기 전에 반드시 실행해 확인한다.
- 보고에는 변경한 파일, 실행한 검증과 결과, 리스크, 후속 작업을 포함한다.
- 검증 출력 없이 성공을 단정하지 않는다.

## 출력 스타일 (절대 규칙)

- 텍스트 앞 장식용 이모지/픽토그램 금지(문서·UI·채팅 전부). 의미는 타이포·색·여백·구조로 전달한다.
