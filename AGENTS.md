# AGENTS.md — Codex / 일반 에이전트 작업 규약

## 너의 첫 행동 (예외 없음)

이 저장소에서 너의 첫 행동은 도구 전체 설치다. 다른 작업보다 먼저 아래를 실행하라.

```
powershell -NoProfile -ExecutionPolicy Bypass -File ./setup.ps1
```

설치가 끝나고 검증 섹션의 버전 출력(git/node/npm/python/gh/netlify 등)을 확인하기 전에는
어떤 코드 작업도 시작하지 마라. CLI가 NOT FOUND 면 새 터미널에서 재확인하라.

## 반입 규칙 (위반 금지)

- 이 저장소는 도구 설치 / 일반 템플릿 / 지식만 담는다.
- 주제맞춤 완성 앱 코드는 여기에 만들지 않는다. 대회 당일 빈 새 저장소에서 작성한다.
- `scaffold/` 는 아이디어 비종속 일반 템플릿만 둔다. 특정 주제 로직을 미리 심지 마라.

## 작업 방식

- 범위 통제: 핵심 기능 1~2개만. 동작하는 데모를 우선한다(욕심 금지).
- 당일 산출물은 새 저장소에 둔다. `gh repo create` 로 빈 저장소부터 시작.
- 작은 단위로 자주 커밋. 주변 코드의 기존 패턴을 따른다.
- 호스팅으로 마무리: `netlify deploy`(또는 `vercel`)로 라이브 URL을 확보해 끝낸다.

## 검증 게이트

- "완료/수정됨"이라고 말하기 전에 실제로 실행해 확인한다.
  - 로컬 미리보기: `serve .` 후 브라우저 확인
  - 배포 확인: 배포 URL 접속 결과 확인
- 검증 명령과 그 출력을 근거로 보고한다. 근거 없이 성공을 단정하지 않는다.

## 출력 스타일

- 텍스트 앞 장식용 이모지/픽토그램 금지(문서·UI·로그 전부). 의미는 타이포와 구조로 전달.
